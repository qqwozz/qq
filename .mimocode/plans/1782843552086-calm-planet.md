# qqwozz-portfolio — Plan улучшений

## Контекст
Персональный портфолио-сайт (React 18 + TS + Vite), задеплоенный на GitHub Pages. Всё в одном `App.tsx` (630 строк) + `styles.css` (1371 строка).

---

## P0 — Критичные баги и мёртвый код

### 1. Дублирующий fetch GitHub API
- **Файл**: `src/App.tsx` ~строки 270-294
- **Проблема**: Два идентичных `fetch('https://api.github.com/users/qqwzz')` — copy-paste баг, тратит лимит GitHub API (60 req/hr без токена)
- **Решение**: Удалить дублирующий блок (второй fetch с теми же обработчиками)

### 2. Удалить неиспользуемые зависимости
- **Файл**: `package.json`
- **Действие**: `npm uninstall gsap @react-three/drei @react-three/fiber three @types/three`
- **Причина**: `gsap` нигде не импортируется (~50KB). Three.js пакеты (~200KB+) используются только в удалённом `MacBookScene.tsx`

### 3. Удалить мёртвые файлы
- `src/components/MacBookScene.tsx` — компонент нигде не импортируется
- `public/MacBook Ultra.glb` (~5MB 3D модель)
- `public/textures/` — 7 PNG файлов текстур
- **Итого**: ~5MB+ мёртвого веса в репозитории

### 4. Исправить бесполезные meta-теги кэширования
- **Файл**: `index.html` строки 5-7
- **Проблема**: `<meta http-equiv="Cache-Control">` не работает — HTTP-кэширование управляется только заголовками ответа. Vite сам добавляет content-hashed имена файлов
- **Решение**: Удалить эти 3 meta-тега

## P1 — Архитектурные улучшения

### 5. Разбить App.tsx на компоненты
- **Текущее состояние**: Весь UI в одном 630-строчном компоненте
- **Решение**: Извлечь секции в отдельные компоненты в `src/components/`:
  - `Navbar.tsx` — навигация
  - `HeroSection.tsx` —.hero блок с анимацией текста
  - `AboutSection.tsx` — о себе + статистика
  - `MarqueeSection.tsx` — бегущая строка технологий
  - `ExperienceSection.tsx` — опыт работы
  - `ProjectsSection.tsx` — проекты
  - `StackSection.tsx` — стек технологий
  - `ContactSection.tsx` — контакты
  - `Footer.tsx` — футер
- **Подход**: Каждый компонент получает `t` (функцию перевода) и данные через props. Стейт `stats` остаётся в `App.tsx` и передаётся в `AboutSection`. Никакого Context/Redux — keep it simple.
- **Результат**: `App.tsx` сократится до ~120 строк (загрузка, layout, стейт)

### 6. Добавить Error Boundary
- **Файл**: Создать `src/components/ErrorBoundary.tsx`
- **Решение**: Простой class-компонент, оборачивающий `<App />` в `main.tsx`. Показывает минимальный fallback UI при краше
- **Причина**: Без него любой runtime error (в Three.js, async fetch, и т.д.) молча убивает весь SPA

### 7. Убрать dangerouslySetInnerHTML
- **Файлы**: `src/App.tsx` строки ~396 и ~575
- **Проблема**: Используется для рендеринга переводов с `<strong>` и `\n` → `<br>`
- **Решение**: Создать хелпер `parseTranslation(text: string): JSX.Element[]` в `src/utils.tsx`, который парсит только `<strong>` и `<br>` — не нужен полный HTML-парсер

### 8. Добавить TypeScript-интерфейсы для API
- **Файл**: Создать `src/types.ts`
- **Интерфейсы**: `GitHubUser`, `LeetcodeStats`, `Stats`
- **Причина**: Сейчас `data` из fetch — не типизированный `any`

### 9. Заменить таймерную загрузку на data-driven
- **Файл**: `src/App.tsx` строки 84-88
- **Проблема**: Загрузка на 600ms + 1000ms — произвольные таймеры, не зависят от реальной загрузки данных/шрифтов
- **Решение**: Показывать загрузку пока `document.fonts.ready` + GitHub fetch не завершатся

## Что НЕ делать
- Не добавлять Redux/Zustand — стейт тривиальный
- Не добавлять React Router — это одна страница
- Не добавлять i18next — текущий хук на 125 строк работает
- Не добавлять Storybook — overkill для 7 секций
- Не дробить CSS на модули — один файл норм для такого размера
- Не добавлять Code Splitting — без Three.js бандл уже крошечный
