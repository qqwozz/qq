# Geo-based Language Switching (RU/EN)

## Goal
Auto-detect visitor's country via IP. Russian IP → Russian text. Any other IP → English text.

## Approach
Lightweight, no external i18n libraries. Use `ip-api.com` (free, CORS-enabled, no API key) on client side.

---

## Files to create/modify

### 1. NEW: `src/i18n.ts` — translations + language hook
- `translations` object with `ru` and `en` keys containing all UI strings
- `useLanguage()` hook that:
  - Calls `http://ip-api.com/json/` on mount
  - If `countryCode === "RU"` → `ru`, else → `en`
  - Falls back to `ru` on network error
  - Returns `{ lang, t }` where `t(key)` looks up the translation
  - Also sets `document.documentElement.lang`

### 2. MODIFY: `src/App.tsx` — use translations
- Import and use `useLanguage()` hook
- Replace all hardcoded Russian strings with `t("key")` calls:
  - Navbar links: "обо мне" → `t("nav.about")`, etc.
  - Hero: name stays Russian (it's a name), subtitle stays "back-end developer"
  - About section: label, paragraph text
  - Stats labels: "репозиториев" → `t("stats.repos")`, etc.
  - Experience: titles, descriptions
  - Projects: descriptions
  - Stack: section title, feature names, skill levels
  - Contact: title
  - Footer: "связаться" link
  - aria-labels, mobile menu
- Also translate text scramble target (name stays same) and hero subtitle

### 3. MODIFY: `index.html` — set default lang
- Change `<html lang="ru">` → `<html lang="ru">` (stays ru as default, will be updated by JS)
